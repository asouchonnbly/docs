---
title: "Description de l’API"
description: "Cette API permet de router des messages unitairement ou par envoi multiple."
---


Elle est mise à disposition sous forme d’une API REST proposant des points d’entrée liés à l’authentification par des identifiants permettant la génération d’un jeton d’accès de type JSON Web Token (JWT) ayant une durée de vie limitée (4H) et permettant l’appel du point d’entrée de
routage.

L’accès à cette API n’est possible qu’en HTTPS (TLS 1.2 minimum).

Les JSON envoyés dans les données de la méthode POST sont validés par le format [JSON schema](https://json-schema.org/). Ces schémas de validation sont donnés dans la documentation à titre informatif.
